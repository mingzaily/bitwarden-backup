## 使用指南

### 1. 配置源服务器

1. 访问 Web 界面 http://localhost:8080
2. 点击"源服务器"标签
3. 点击"添加服务器"按钮
4. 填写服务器信息：
   - **名称**: 服务器标识名称
   - **服务器 URL**: Bitwarden 服务器地址（如 `https://vault.bitwarden.com`）
   - **Client ID**: API 客户端 ID
   - **Client Secret**: API 客户端密钥
   - **Master Password**: 主密码（用于解锁密码库）
   - **官方服务器**: 勾选表示这是 Bitwarden 官方服务器

> **为什么需要主密码？** Bitwarden CLI 使用 API Key 登录后，仍需要主密码来解锁密码库才能导出数据。

### 2. 配置备份目标

1. 点击"备份目标"标签
2. 点击"添加目标"按钮
3. 选择目标类型：
   - **本地存储**: 备份到本地目录（如 `/app/backups`）
   - **WebDAV**: 备份到 WebDAV 服务器（如 Nextcloud、坚果云）
   - **目标服务器**: 导入到另一个 Bitwarden 服务器

### 3. 创建备份任务

1. 点击"备份任务"标签
2. 点击"创建任务"按钮
3. 配置任务参数：
   - **任务名称**: 任务标识名称
   - **源服务器**: 选择要备份的服务器
   - **Cron 表达式**: 定时规则
   - **备份目标**: 选择一个或多个备份目标
   - **启用任务**: 是否启用此任务

### 4. Cron 表达式说明

格式：`秒 分 时 日 月 周`

常用示例：
- `0 0 2 * * *` - 每天凌晨 2 点
- `0 0 */6 * * *` - 每 6 小时
- `0 30 1 * * 1` - 每周一凌晨 1:30
- `0 0 0 1 * *` - 每月 1 号凌晨

### 5. 查看执行日志

1. 点击"执行日志"标签
2. 查看备份任务的执行历史
3. 状态说明：
   - **success**: 执行成功
   - **failed**: 执行失败
   - **running**: 正在执行

### 6. 注意事项

- 备份到 WebDAV 或目标服务器后，临时文件会自动清理
- 本地备份路径建议使用绝对路径
- 确保 Bitwarden CLI 已正确安装并可用
